Hooks are Fraxel's primary tool for connecting JSX components to the engine's event system, reactive state, and node hierarchy. If you've used React hooks before, the mental model is familiar — functions that register behavior during a component's setup phase. In Fraxel, hooks run when the component function executes (before the JSX is returned) and are automatically cleaned up when the node is destroyed.

## The Golden Rule

Call hooks at the **top level of a component function**, never inside a callback, `if` statement, or loop:

```tsx
// ✅ Correct — called at component scope
function Player() {
  const sprite = useSprite()
  const [health, setHealth] = useSignal(100)
  useEffect(() => { /* ... */ })
  return <sprite ref={sprite} textureId={PLAYER} />
}

// ❌ Wrong — called inside a callback
function Player() {
  const handleClick = () => {
    const sprite = useSprite() // ERROR: wrong scope
  }
  return <sprite textureId={PLAYER} />
}
```

## `useSprite` \+ `useTrigger`

These two hooks form the foundation of node interaction. `useSprite` creates a typed reference that you attach to a JSX element with `ref={}`. The reference exposes each of the node's underlying `Event`s as a `Trigger` — for example, `updated`, `destroyed`, `colliderEntered` — so you can pass them to `useTrigger` and get automatic cleanup on destroy:

```tsx enemy.tsx
import { useCollider, useRigidBody, useTrigger } from 'fraxel/hooks'
import { shapes } from 'fraxel'

function Enemy() {
  const collider = useCollider()
  const body = useRigidBody()

  useTrigger(collider.colliderEntered, (other) => {
    if (other.node.name === 'bullet') {
      body.node.destroy()
    }
  })

  return (
    <rigid-body ref={body} position={[300, 200]}>
      <sprite textureId={ENEMY} />
      <collider
        ref={collider}
        shape={shapes.circle(16)}
        group={['enemy']}
        collidesWith={['bullet']}
      />
    </rigid-body>
  )
}
```

`useTrigger` subscriptions are automatically removed when the node is destroyed.

## `useMount`

Run code once when the node starts, with optional cleanup on destroy:

```tsx fade-in.tsx
import { useMount, useSprite } from 'fraxel/hooks'
import { tween } from 'fraxel'

function FadeIn() {
  const sprite = useSprite()

  useMount(() => {
    tween({ target: sprite.node, prop: 'opacity', from: 0, to: 1, duration: 0.5 })

    // Return a cleanup function (runs on destroy)
    return () => {
      console.log('Node cleaned up')
    }
  })

  return <sprite ref={sprite} textureId={TEX} />
}
```

The cleanup function is optional. Use it to cancel intervals, clear external state, or stop tweens.

## `ref.node.child`

Access a specific child node anywhere in the subtree by providing a path string (dot-separated node names) and the expected type:

```tsx
import { useSprite, useMount } from 'fraxel/hooks'

function Player() {
  // Access the sprite named 'body' that's a child of 'visual'
  const visual = useSprite()
  const bodySprite = useComputed(() => visual.node.child('body', PrimaryNode.Sprite))

  useMount(() => {
    bodySprite.node.brightness = 1.5
  })

  return (
    <transform>
      <transform name="visual" ref={visual}>
        <sprite name="body" textureId={PLAYER} />
      </transform>
    </transform>
  )
}
```

## `useRef`

<Warning>
  `useRef` is deprecated. Use a plain `let` variable instead. Component functions in Fraxel execute only once, so a `let` at the top of the function body persists across frames without triggering reactive updates.
</Warning>

Store a mutable value that persists across renders without triggering reactive updates. Unlike `useSignal`, changing a ref's value doesn't re-run effects or recompute derived signals:

```tsx
// Before (deprecated)
const lastAttackTime = useRef(0)

// After — use a plain let
let lastAttackTime = 0

function Player() {
  const attack = () => {
    const now = performance.now()
    if (now - lastAttackTime > 500) { // 500ms cooldown
      lastAttackTime = now
      // Fire bullet...
    }
  }

  return <transform>{/* ... */}</transform>
}
```

## `useScene`

Access the top-level scene controls from any component:

```tsx
import { useScene, useClickable, useTrigger } from 'fraxel/hooks'

function GameOverScreen() {
  const scene = useScene()
  const btn = useClickable()

  useTrigger(btn.clicked, () => {
    scene.change('level1')
  })

  return (
    <sprite textureId={RETRY_BTN}>
      <clickable ref={btn} size={[120, 40]} />
    </sprite>
  )
}
```

See [Scenes](/guides/scenes) for all the scene control methods.

## `useTrigger` — Cross-Component Pub/Sub

When components need to communicate without sharing props or a parent-child relationship, use triggers. Create a trigger at module scope with `createTrigger`, subscribe in one component with `useTrigger`, and emit from another:

```tsx triggers.ts
import { createTrigger } from 'fraxel/hooks'

export const onPlayerDied = createTrigger<[]>()
export const onScoreChanged = createTrigger<[number]>()
```

```tsx player.tsx
import { onPlayerDied } from './triggers.js'

function Player() {
  const takeDamage = (amount: number) => {
    // ...
    if (health <= 0) {
      onPlayerDied.emit()
    }
  }
  return <transform>{/* ... */}</transform>
}
```

```tsx hud.tsx
import { useTrigger } from 'fraxel/hooks'
import { onPlayerDied } from './triggers.js'

function HUD() {
  useTrigger(onPlayerDied, () => {
    // Show game over UI
  })
  return <transform>{/* ... */}</transform>
}
```

`useTrigger` subscriptions clean up automatically on destroy.

## `createContext` / `useContext`

Share data down a component tree without prop drilling. Create a context with a default value, wrap children in the `Provider`, and read it anywhere below with `useContext`:

```tsx game-context.tsx
import { createContext, useContext } from 'fraxel/hooks'

interface GameCtxValue {
  difficulty: 'easy' | 'normal' | 'hard'
  score: SignalGetter<number>
}

export const GameCtx = createContext<GameCtxValue>({
  difficulty: 'normal',
  score: () => 0,
})
```

```tsx game-scene.tsx
import { GameCtx } from './game-context.js'

function GameScene() {
  const [score] = useSignal(0)

  return (
    <GameCtx.Provider value={{ difficulty: 'hard', score }}>
      <Player />
      <HUD />
      <EnemySpawner />
    </GameCtx.Provider>
  )
}
```

```tsx hud.tsx
import { GameCtx } from './game-context.js'

function HUD() {
  const { difficulty, score } = useContext(GameCtx)

  return (
    <text
      position={[240, 16]}
      text={() => `[${difficulty.toUpperCase()}] Score: ${score()}`}
      style={{ fontSize: 14, foregroundColor: '#fff', textAlign: 'center' }}
    />
  )
}
```

### `useCondition`

Creates a reactive boolean that flips between `true` and `false` based on two opposing events:

```tsx
// true while the raycast detects an enemy
const isDetecting = useCondition(rayCast, 'colliderEntered', 'colliderExited')

// true while the player is hovering over a button
const isHovered = useCondition(clickable, 'mouseEntered', 'mouseExited')
```

### `useTimer`

Wraps `<timer>` with reactive progress and control methods:

```tsx cooldown.tsx
import { useTimer } from 'fraxel/hooks'
import { shapes } from 'fraxel'

function Cooldown() {
  const timer = useTimer()

  const barColor = useComputed(() =>
    timer.progress() >= 1 ? ([0.2, 0.9, 0.2, 1] as const) : ([0.9, 0.4, 0.1, 1] as const)
  )

  return (
    <transform>
      <timer ref={timer} duration={3} />
      <geometry
        shape={() => shapes.rectangle(progress() * 80, 8)}
        fillColor={barColor}
      />
    </transform>
  )
}
```

### `useClickable`

Returns a node ref and a reactive `hovered` boolean:

```tsx button.tsx
import { useClickable, useTrigger, useComputed } from 'fraxel/hooks'

function Button({ onClick }: { onClick: () => void }) {
  const btn = useClickable()
  const [hovered, setHovered] = useSignal(false)

  useTrigger(btn.mouseEntered, () => setHovered(true))
  useTrigger(btn.mouseExited, () => setHovered(false))

  const scale = useComputed(() => (hovered() ? 1.05 : 1.0))

  useTrigger(btn.clicked, onClick)

  return (
    <sprite textureId={BTN} displaySize={() => [64 * scale(), 24 * scale()]}>
      <clickable ref={btn} size={[64, 24]} />
    </sprite>
  )
}
```

### `useMatch` and `useWhen`

Convenience hooks for conditional signal values:

```tsx
// useMatch — like a switch expression on a signal
const animName = useMatch(playerState, {
  idle: 'idle',
  walk: 'walk',
  jump: 'jump',
  attack: 'attack',
})

// useWhen — ternary on a boolean signal
const brightness = useWhen(isHovered, 1.3, 1.0)
```
