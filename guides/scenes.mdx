A complete game is more than one screen. You need a main menu, a gameplay level, a pause screen, a game-over state. Scenes let you divide your game into discrete, self-contained units that Fraxel loads and unloads on demand. Each scene is a separate JSX tree — when you switch scenes, the old tree is destroyed and the new one starts fresh.

## Defining Scenes

You define scenes declaratively when you call `createGame`. Pass a `<Game>` element containing one `<Scene>` element per scene, each with a unique `name` and a `component` function:

```tsx main.tsx
import { createGame, Game, Scene } from 'fraxel/jsx'
import { loadTexture } from 'fraxel/assets'

const game = createGame(
  <Game width={480} height={270} defaultScene="menu">
    <Scene name="menu" component={() => <MainMenu />} />
    <Scene name="level1" component={() => import('./scenes/level1.js')} />
    <Scene name="gameover" component={() => <GameOver />} />
  </Game>,
  document.querySelector('#root')!,
)

game.play()
```

The `defaultScene` prop on `<Game>` sets which scene loads first. Each `<Scene>` component prop can be:

- A plain function returning JSX: `() => <MyScene />`
- An async function returning JSX: `async () => { const t = await loadTexture(…); return <MyScene tex={t} /> }`
- A dynamic import returning a module with a default export: `() => import('./scenes/level1.js')`

## Switching Scenes

Use `useScene()` inside any component to access scene controls. Call `changeScene` with a registered scene name to transition:

```tsx MainMenu.tsx
import { useScene, useClickable, useTrigger } from 'fraxel/hooks'

function MainMenu() {
  const scene = useScene()
  const btn = useClickable()

  useTrigger(btn.clicked, () => {
    scene.change('level1')
  })

  return (
    <transform>
      <text
        position={[240, 100]}
        text="SPACE SHOOTER"
        style={{ fontSize: 32, foregroundColor: '#fff', textAlign: 'center' }}
      />
      <sprite textureId={PLAY_BTN}>
        <clickable ref={btn} size={[120, 40]} />
      </sprite>
    </transform>
  )
}
```

`changeScene` destroys the current scene's node tree before loading the new one. All signals, hooks, and event subscriptions in the old scene are cleaned up automatically.

## Preloading Scenes

For fast scene transitions — especially when a scene involves heavy asset loading — preload it while the current scene is still running. `preloadScene` returns a `Promise` that resolves to an activation function you call to swap the scene instantly:

```tsx Level1.tsx
import { useScene, useMount } from 'fraxel/hooks'

function Level1() {
  const scene = useScene()
  let activateLevel2: (() => void) | null = null

  useMount(async () => {
    // Start loading level 2 in the background immediately
    activateLevel2 = await scene.preloadScene('level2')
  })

  const onLevelComplete = () => {
    // By the time the player finishes level 1, level 2 is likely ready
    activateLevel2?.()
  }

  return (
    <transform>
      {/* level 1 content */}
    </transform>
  )
}
```

<Note>
  `preloadScene` runs the scene's component function in the background. The returned activation function swaps the scene without re-loading — the JSX tree is already built.
</Note>

## `useScene` Hook

`useScene` gives any component access to the top-level scene controls:

```tsx PauseButton.tsx
import { useScene, useClickable, useTrigger } from 'fraxel/hooks'

function PauseButton() {
  const scene = useScene()
  const btn = useClickable()

  useTrigger(btn.clicked, () => {
    scene.pause()
  })

  return (
    <sprite textureId={PAUSE_BTN}>
      <clickable ref={btn} size={[32, 32]} />
    </sprite>
  )
}
```

| Method | Description |
|--------|-------------|
| `scene.change(name)` | Destroy the current scene and load a new one |
| `scene.preload(name)` | Load a scene in the background; returns a `Promise<() => void>` activation function |
| `scene.current` | The name of the currently active scene |

## Per-Scene Asset Loading

A scene's `component` function is the natural place for scene-specific asset loading. Because it supports `async/await`, you can load exactly what the scene needs before the JSX is returned:

```tsx main.tsx
import { createGame, Game, Scene } from 'fraxel/jsx'
import { loadTexture, loadBatch } from 'fraxel/assets'

const game = createGame(
  <Game width={480} height={270} defaultScene="boss-fight">
    <Scene
      name="boss-fight"
      component={async () => {
        const [bossTexture, arenaTexture] = await loadBatch([
          () => loadTexture('/assets/boss.png'),
          () => loadTexture('/assets/arena.png'),
        ])
        return <BossFight boss={bossTexture} arena={arenaTexture} />
      }}
    />
  </Game>,
  document.querySelector('#root')!,
)

game.play()
```

For memory management, unload scene-specific assets when leaving:

```tsx BossFight.tsx
import { useMount } from 'fraxel/hooks'
import { unloadTexture } from 'fraxel/assets'

function BossFight({ boss, arena }: BossFightProps) {
  useMount(() => {
    // Return a cleanup function — runs when the scene node is destroyed
    return () => {
      unloadTexture(boss)
      unloadTexture(arena)
    }
  })

  return <transform>{/* boss fight content */}</transform>
}
```

## Loading Screen Pattern

Show a loading screen while the next scene loads by switching to a lightweight loading scene first, then transitioning once the heavy scene is ready:

```tsx main.tsx
import { createGame, Game, Scene } from 'fraxel/jsx'

const game = createGame(
  <Game width={480} height={270} defaultScene="loading">
    <Scene name="loading" component={() => <LoadingScreen />} />
    <Scene name="level3" component={() => import('./scenes/level3.js')} />
  </Game>,
  document.querySelector('#root')!,
)

game.play()
```

```tsx LoadingScreen.tsx
import { useScene, useMount } from 'fraxel/hooks'

function LoadingScreen() {
  const scene = useScene()

  useMount(async () => {
    // Begin loading the real scene, then switch when ready
    const activate = await scene.preloadScene('level3')
    activate()
  })

  return (
    <transform>
      <text
        position={[240, 135]}
        text="Loading…"
        style={{ fontSize: 24, foregroundColor: '#fff', textAlign: 'center' }}
      />
    </transform>
  )
}
```
