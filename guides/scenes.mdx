A complete game is more than one screen. You need a main menu, a gameplay level, a pause screen, a game-over state. Scenes let you divide your game into discrete, self-contained units that Fraxel loads and unloads on demand. Each scene is a component that returns one JSX tree.

## Defining Scenes

The generated `src/main.tsx` already calls `createGame()`. Add one `<SceneRoot>` per scene, each with a unique `name` and a `component` function:

```tsx main.tsx
import { createGame, GameRoot, SceneRoot } from 'fraxel'

const game = createGame(
  <GameRoot width={480} height={270} defaultScene="menu">
    <SceneRoot name="menu" component={() => <MainMenu />} />
    <SceneRoot name="level1" component={() => import('./scenes/level1.js')} />
    <SceneRoot name="gameover" component={() => <GameOver />} />
  </GameRoot>,
  document.querySelector('#app')!,
)

game.play()
```

The `defaultScene` prop on `<GameRoot>` sets which scene loads first. Each `<SceneRoot>` component prop can be:

- A plain function returning JSX: `() => <MyScene />`
- A dynamic import returning a module with a default export: `() => import('./scenes/level1.js')`

## Switching Scenes

Use `useScene()` inside any component to access scene controls. Call `change` with a registered scene name to transition:

```tsx main-menu.tsx
import { useScene, useClickable, useTrigger } from 'fraxel'

function MainMenu() {
  const scene = useScene()
  const btn = useClickable()

  useTrigger(btn.clicked, () => {
    scene.change('level1')
  })

  return (
    <transform>
      <text
        position={[240, 100]}
        text="SPACE SHOOTER"
        style={{ fontSize: 32, foregroundColor: '#fff', textAlign: 'center' }}
      />
      <sprite textureId={PLAY_BTN}>
        <clickable ref={btn} size={[120, 40]} />
      </sprite>
    </transform>
  )
}
```

`change` destroys the current scene's node tree before loading the new one. All signals, hooks, and event subscriptions in the old scene are cleaned up automatically.

## Preloading Scenes

For fast scene transitions, preload a scene while the current scene is still running. `preload` returns a `Promise` that resolves to an activation function you call to swap the scene instantly:

```tsx level1.tsx
import { useScene, useMount } from 'fraxel'

function Level1() {
  const scene = useScene()
  let activateLevel2: (() => void) | null = null

  useMount(async () => {
    // Start loading level 2 in the background immediately
    activateLevel2 = await scene.preload('level2')
  })

  const onLevelComplete = () => {
    // By the time the player finishes level 1, level 2 is likely ready
    activateLevel2?.()
  }

  return (
    <transform>
      {/* level 1 content */}
    </transform>
  )
}
```

<Note>
  `preload` runs the scene's component function in the background. The returned activation function swaps the scene without re-loading — the JSX tree is already built.
</Note>

## `useScene` Hook

`useScene` gives any component access to the top-level scene controls:

```tsx pause-button.tsx
import { useScene, useClickable, useTrigger } from 'fraxel'

function PauseButton() {
  const scene = useScene()
  const btn = useClickable()

  useTrigger(btn.clicked, () => {
    scene.change('pause')
  })

  return (
    <sprite textureId={PAUSE_BTN}>
      <clickable ref={btn} size={[32, 32]} />
    </sprite>
  )
}
```

| Method | Description |
|--------|-------------|
| `scene.change(name)` | Destroy the current scene and load a new one |
| `scene.preload(name)` | Load a scene in the background; returns a `Promise<() => void>` activation function |
| `scene.getCurrent()` | The name of the currently active scene |

## Scene-specific assets

For static scene assets, import them inside the scene module with `?texture` or `?sound`:

```tsx src/scenes/boss-fight.tsx
import BOSS from '../assets/boss.png?texture'
import ARENA from '../assets/arena.png?texture'

export default function BossFight() {
  return (
    <transform>
      <sprite textureId={ARENA} />
      <sprite textureId={BOSS} />
    </transform>
  )
}
```

Use runtime loaders only when the URL is not known at build time. See [Asset Pipeline](/guides/asset-pipeline) for runtime loading and unloading patterns.

## Loading Screen Pattern

Show a loading screen while the next scene loads by switching to a lightweight loading scene first, then transitioning once the heavy scene is ready:

```tsx main.tsx
import { createGame, GameRoot, SceneRoot } from 'fraxel'

const game = createGame(
  <GameRoot width={480} height={270} defaultScene="loading">
    <SceneRoot name="loading" component={() => <LoadingScreen />} />
    <SceneRoot name="level3" component={() => import('./scenes/level3.js')} />
  </GameRoot>,
  document.querySelector('#app')!,
)

game.play()
```

```tsx loading-screen.tsx
import { useScene, useMount, useTrigger } from 'fraxel'
import { onLost } from '../triggers.js'

function LoadingScreen() {
  const scene = useScene()

  useMount(async () => {
    // Begin loading the real scene, then switch when ready
    const activate = await scene.preload('level3')
    activate()
  })

  useTrigger(onLost, () => {
    scene.change(scene.getCurrent()) // reload scene
  })

  return (
    <transform>
      <text
        position={[240, 135]}
        text="Loading…"
        style={{ fontSize: 24, foregroundColor: '#fff', textAlign: 'center' }}
      />
    </transform>
  )
}
```
