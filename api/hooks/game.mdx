Game hooks provide control over the game loop, pause state, browser focus events, and scene management from inside components.

---

## useGame

Returns imperative control over the game loop and triggers for browser focus events.

```ts
useGame(): {
  play: () => void
  stop: () => void
  destroy: () => void
  blurred: Trigger<[]>
  focused: Trigger<[]>
}
```

**Returns** `PausedControllers`:

| Property | Type | Description |
|---|---|---|
| `play` | `() => void` | Starts the game loop. |
| `stop` | `() => void` | Stops the animation-frame loop without destroying the game. |
| `destroy` | `() => void` | Destroys the game and unmounts all nodes. |
| `blurred` | `Trigger<[]>` | Fires whenever the browser window loses focus. |
| `focused` | `Trigger<[]>` | Fires whenever the browser window gains focus. |

```tsx
import { useGame, useTrigger } from 'fraxel'
import { Input } from 'fraxel'

function GameController() {
  const { play, stop, blurred, focused } = useGame()

  useTrigger(blurred, () => { console.log('lost focus') })
  useTrigger(focused, () => { console.log('won focus') })

  return <clickable onClick={() => stop()} size={[64, 32]} />
}
```

## usePaused

Returns reactive access to the game's pause state and imperative pause/resume controls.

```ts
usePaused(): { paused: SignalGetter<boolean>, resume: () => void, pause: () => void }
```

**Returns** `PausedControllers`:

| Property | Type | Description |
|---|---|---|
| `paused` | `SignalGetter<boolean>` | Reactive signal that is `true` when the game is paused. |
| `resume` | `() => void` | Clears the pause state. |
| `pause` | `() => void` | Sets the pause state. The loop keeps rendering, but updates and physics are skipped. |

```tsx
import { usePaused, useEffect } from 'fraxel'
import { Input } from 'fraxel'

const Pause = Input.createAction({ key: 'p' })

function PauseManager() {
  const { paused, resume, pause } = usePaused()

  useEffect(() => {
    if (Input.justActionPressed(Pause)) {
      paused() ? resume() : pause()
    }
  })

  return null
}
```

## useScene

Returns imperative access to the scene manager for switching and preloading scenes.

```ts
useScene(): { getCurrent: () => string, change: (sceneName: string | null) => Promise<void>, preload: (scene: string) => Promise<() => void> }
```

**Returns** `SceneControllers`:

| Property | Type | Description |
|---|---|---|
| `getCurrent` | `() => string` | Returns the name of the currently active scene. |
| `change` | `(sceneName: string \| null) => Promise<void>` | Destroys the current scene and loads the named one. Pass `null` to unload without switching. |
| `preload` | `(scene: string) => Promise<() => void>` | Preloads a scene in the background. Returns a function to switch to the preloaded scene. |

```tsx
import { useScene, useEffect } from 'fraxel'
import { Input } from 'fraxel'

const Restart = Input.createAction({ key: 'r' })

function GameOver() {
  const scene = useScene()

  useEffect(() => {
    if (Input.justActionPressed(Restart)) {
      scene.change('game')
    }
  })

  return <text text="Game Over" />
}
```

## Dynamic spawning

Use the `spawn(jsx)` method on any node reference to dynamically append child nodes at runtime. This is the recommended way to spawn entities from within a component.

```tsx
import { useTransform, useClickable, useTrigger } from 'fraxel'

function Spawner() {
  const container = useTransform()
  const btn = useClickable()

  useTrigger(btn.clicked, () => {
    container.spawn(<Enemy position={[Math.random() * 200, 0]} />)
  })

  return (
    <transform ref={container}>
      <clickable ref={btn} size={[64, 32]} />
    </transform>
  )
}
```
