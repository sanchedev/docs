The `Game` class is the static singleton that owns the canvas, drives the game loop, and manages scenes. In most projects you interact with it indirectly through `createGame` and the `GameControls` object it returns — but the static API is available for advanced use cases such as custom loop control or imperative scene switching outside a component.

---

## createGame

The recommended way to bootstrap a Fraxel application. Parses a `<GameRoot>` JSX tree, sets up the engine, registers all `<SceneRoot>` children, loads the default scene, and returns `GameControls`.

```ts
createGame(jsx: Fraxel.Node, root: HTMLElement): GameControls
```

<ParamField path="jsx" type="Fraxel.Node" required>
  A `<GameRoot>` element containing one or more `<SceneRoot>` children.
</ParamField>
<ParamField path="root" type="HTMLElement" required>
  The DOM element that will become the parent of the generated `<canvas>`. Must not be `null`.
</ParamField>

**Returns** [`GameControls`](#gamecontrols).

**Throws:**
- `MissingGameRootError` — `root` is `null` or `undefined`.
- `InvalidGameElementError` — the top-level JSX element is not `<GameRoot>`.
- `MissingSceneError` — a child element is not `<SceneRoot>`.
- `InvalidSceneComponentError` — the scene component did not return exactly one root node.

```tsx
import { createGame, GameRoot, SceneRoot } from 'fraxel'

const game = createGame(
  <GameRoot width={320} height={180} defaultScene="main">
    <SceneRoot name="main"  component={() => import('./scenes/main.js')} />
    <SceneRoot name="menu"  component={() => import('./scenes/menu.js')} />
  </GameRoot>,
  document.querySelector('#root')!
)

game.play()
```

<Note>
`createGame` automatically calls `Game.setup()`, registers all scenes with `SceneManager`, and loads `defaultScene`. You do not need to call `Game.setup()` separately.
</Note>

## GameControls

The object returned by `createGame`. Provides the primary API for controlling the running game.

```ts
interface GameControls {
  play():                          void
  pause():                         void
  destroy():                       void
}
```

| Property | Type | Description |
|---|---|---|
| `paused` | `SignalGetter<boolean>` | Reactive signal that is `true` when the game loop is paused. Read with `game.paused()`. |
| `play()` | `() => void` | Starts (or resumes) the game loop. Acquires a screen wake-lock if available. |
| `pause()` | `() => void` | Pauses the game loop. Call `play()` to resume. |
| `destroy()` | `() => void` | Stops the loop, tears down the current scene, destroys the input system, and releases all resources. |
| `changeScene(name)` | `(name: string) => Promise<void>` | Destroys the current scene, loads the named scene, and resolves when the transition is complete. |
| `preloadScene(name)` | `(name: string) => Promise<() => void>` | Loads a scene in the background without switching to it. Resolves with a function you can call later to perform the instant switch. |
| `getSize()` | `() => Vector2` | Returns a `Vector2` with the canvas's logical pixel dimensions (as configured in `<GameRoot width height>`). |

<CodeGroup>
```ts Basic lifecycle
const game = createGame(<GameRoot ...>, root)

game.play()

// Later — e.g. on window unload
window.addEventListener('unload', () => game.destroy())
```
```ts Scene transitions
const game = createGame(<GameRoot ...>, root)
game.play()

// Switch immediately
await SceneManager.changeScene('level-2')

// Or preload first, then switch instantly
const activate = await SceneManager.preloadScene('boss-fight')
// ... show a loading bar, then:
activate()
```
</CodeGroup>

## Game (static class)

The underlying static singleton. `createGame` wraps this, but you can use it directly when you need imperative control outside the JSX bootstrap path.

### Game.setup

Initialises the canvas, input system, and renderer. Must be called before `play()`.

```ts
Game.setup(options: SetupOptions): void
```

<ParamField path="options.width" type="number" required>
  Logical canvas width in pixels.
</ParamField>
<ParamField path="options.height" type="number" required>
  Logical canvas height in pixels.
</ParamField>
<ParamField path="options.root" type="HTMLElement" required>
  Parent DOM element for the generated canvas.
</ParamField>
<ParamField path="options.pauseOnBlur" type="boolean">
  When `true`, the game pauses on browser blur (tab switch, minimize) and requires a manual `play()` call to resume. Defaults to `false`.
</ParamField>
<ParamField path="options.testOptions" type="Partial<TestOptions>">
  Debug overlays. See [`TestOptions`](#testoptions).
</ParamField>
<ParamField path="options.theme" type="Theme">
  Custom render theme. Defaults to `new Theme()`.
</ParamField>
<ParamField path="options.pixelated" type="boolean">
  When `true`, disables canvas image rendering — sets `imageSmoothingEnabled = false` on the canvas context. Use for pixel-art games to prevent blurry upscaling of sprite textures. Defaults to `false`.
</ParamField>

```ts
import { Game } from 'fraxel'

Game.setup({
  width:  480,
  height: 270,
  root:   document.querySelector('#root')!,
  testOptions: { showColliders: true },
  pixelated: true,
})
```

### Game.play

Starts the animation-frame loop. Throws `EngineNotSetupError` if called before `Game.setup`.

```ts
Game.play(): void
```

### Game.resume

Resume the loop. The game changes mode `PLAYING`.

```ts
Game.resume(): void
```

### Game.pause

Pauses the loop. The game changes mode `PAUSED`.

```ts
Game.pause(): void
```

### Game.stop

Pauses the loop. The canvas freezes on the last rendered frame.

```ts
Game.stop(): void
```

### Game.destroy

Cancels the animation frame, tears down input listeners, destroys the active scene, and resets setup state.

```ts
Game.destroy(): void
```

### Game.isRunning

A signal of the current game status mode (`true` / `false`). If it is `true` that is because the game was initialized and it is playing. If it is `false` that means the game is frreze or was destroyed.

```ts
Game.isRunning: SignalGetter<boolean>
```

### Game.isPaused

A signal of the current pause mode (`true` / `false`).

```ts
Game.isPaused: SignalGetter<boolean>
```

## Game.blurred

An `Event` that fires whenever the browser window loses focus (tab switch, minimize, etc.). The game is automatically paused when this fires.

```ts
Game.blurred: Event<[], 'blur'>
```

```ts
import { Game } from 'fraxel'

Game.blurred.on(() => {
  console.log('Game paused — tab lost focus')
  showPauseOverlay()
})
```

<Note>
The game auto-pauses on blur and resumes when the player calls `game.play()` again. You can hook into `Game.blurred` to show a pause screen.
</Note>

## Game.focused

An `Event` that fires whenever the browser window is focus. The game automatically continue when this fires.

```ts
Game.focused: Event<[], 'focus'>
```

## TestOptions

Debug rendering options passed to `Game.setup` or `<GameRoot testOptions={...}>`.

```ts
interface TestOptions {
  showColliders:  boolean  // Draw collider shapes — default false
  showRayCasts:   boolean  // Draw ray cast lines — default false
  showClickables: boolean  // Draw clickable areas — default false
}
```

All properties default to `false`. Enable them during development to visualise invisible game objects:

```ts
Game.setup({
  width: 320, height: 180, root,
  testOptions: {
    showColliders:  true,
    showRayCasts:   true,
    showClickables: true,
  },
})
```
