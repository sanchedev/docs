The `Game` class is the static singleton that owns the canvas, drives the game loop, and manages scenes. In most projects you interact with it indirectly through `createGame` and the `GameControls` object it returns ‚Äî but the static API is available for advanced use cases such as custom loop control or imperative scene switching outside a component.

---

## createGame

The recommended way to bootstrap a Fraxel application. Parses a `<GameRoot>` JSX tree, sets up the engine, registers all `<SceneRoot>` children, loads the default scene, and returns `GameControls`.

```ts
createGame(jsx: Fraxel.Node, root: HTMLElement): GameControls
```

<ParamField path="jsx" type="Fraxel.Node" required>
  A `<GameRoot>` element containing one or more `<SceneRoot>` children.
</ParamField>
<ParamField path="root" type="HTMLElement" required>
  The DOM element that will become the parent of the generated `<canvas>`. Must not be `null`.
</ParamField>

**Returns** [`GameControls`](#gamecontrols).

**Throws:**
- `MissingGameRootError` ‚Äî `root` is `null` or `undefined`.
- `InvalidGameElementError` ‚Äî the top-level JSX element is not `<GameRoot>`.
- `MissingSceneError` ‚Äî a child element is not `<SceneRoot>`.
- `InvalidSceneComponentError` ‚Äî the scene component did not return exactly one root node.

```tsx
import { createGame, GameRoot, SceneRoot } from 'fraxel'

const game = createGame(
  <GameRoot width={320} height={180} defaultScene="main">
    <SceneRoot name="main"  component={() => import('./scenes/main.js')} />
    <SceneRoot name="menu"  component={() => import('./scenes/menu.js')} />
  </GameRoot>,
  document.querySelector('#app')!
)

game.play()
```

<Note>
`createGame` automatically calls `Game.setup()`, registers all scenes with `SceneManager`, and loads `defaultScene`. You do not need to call `Game.setup()` separately.
</Note>

## GameControls

The object returned by `createGame`. Provides the primary API for controlling the running game.

```ts
interface GameControls {
  play(): void
  pause(): void
  destroy(): void
}
```

| Method | Type | Description |
|---|---|---|
| `play()` | `() => void` | Starts the game loop. |
| `pause()` | `() => void` | Sets the game pause state to paused. Rendering continues, but updates and physics are skipped. |
| `destroy()` | `() => void` | Stops the loop, destroys the current scene, destroys the input system, and removes engine listeners. |

`GameControls` intentionally stays small. Use `SceneManager` for scene transitions and the static `Game.isPaused` / `Game.isRunning` signals for state.

<CodeGroup>
```ts Basic lifecycle
const game = createGame(<GameRoot ...>, root)

game.play()

// Later ‚Äî e.g. on window unload
window.addEventListener('unload', () => game.destroy())
```
```ts Scene transitions
const game = createGame(<GameRoot ...>, root)
game.play()

// Switch immediately
await SceneManager.setScene('level-2')

// Or preload first, then switch instantly
const activate = await SceneManager.preloadScene('boss-fight')
// ... show a loading bar, then:
activate()
```
</CodeGroup>

## Game (static class)

The underlying static singleton. `createGame` wraps this, but you can use it directly when you need imperative control outside the JSX bootstrap path.

### Game.setup

Initialises the canvas, input system, and renderer. Must be called before `play()`.

```ts
Game.setup(options: SetupOptions): void
```

<ParamField path="options.width" type="number" required>
  Logical canvas width in pixels.
</ParamField>
<ParamField path="options.height" type="number" required>
  Logical canvas height in pixels.
</ParamField>
<ParamField path="options.root" type="HTMLElement" required>
  Parent DOM element for the generated canvas.
</ParamField>
<ParamField path="options.pauseOnBlur" type="boolean">
  When `true`, the game pauses on browser blur (tab switch, minimize) and requires a manual `play()` call to resume. Defaults to `false`.
</ParamField>
<ParamField path="options.testOptions" type="Partial<TestOptions>">
  Debug overlays. See [`TestOptions`](#testoptions).
</ParamField>
<ParamField path="options.theme" type="Theme">
  Custom render theme. Defaults to `new Theme()`.
</ParamField>
<ParamField path="options.pixelated" type="boolean">
  When `true`, disables canvas image rendering ‚Äî sets `imageSmoothingEnabled = false` on the canvas context. Use for pixel-art games to prevent blurry upscaling of sprite textures. Defaults to `false`.
</ParamField>

```ts
import { Game } from 'fraxel'

Game.setup({
  width:  480,
  height: 270,
  root:   document.querySelector('#app')!,
  testOptions: { showColliders: true },
  pixelated: true,
})
```

### Game.play

Starts the animation-frame loop. Throws `EngineNotSetupError` if called before `Game.setup`.

```ts
Game.play(): void
```

### Game.resume

Clears the pause state. The animation-frame loop must already be running for updates to resume.

```ts
Game.resume(): void
```

### Game.pause

Sets the pause state. The loop keeps rendering, but scene updates, collision updates, and physics updates are skipped.

```ts
Game.pause(): void
```

### Game.stop

Stops the animation-frame loop entirely. This is separate from `Game.pause()` and does not reset `Game.isPaused`.

```ts
Game.stop(): void
```

### Game.destroy

Cancels the animation frame, tears down input listeners, destroys the active scene, and resets setup state.

```ts
Game.destroy(): void
```

### Game.isRunning

A signal that indicates whether the animation-frame loop is running.

```ts
Game.isRunning: SignalGetter<boolean>
```

### Game.isPaused

A signal of the current pause mode (`true` / `false`).

```ts
Game.isPaused: SignalGetter<boolean>
```

## Game.blurred

An `Event` that fires whenever the browser window loses focus (tab switch, minimize, etc.). The game loop is stopped on blur. If `pauseOnBlur` is enabled, the pause state is also set to `true`.

```ts
Game.blurred: Event<[], 'blur'>
```

```ts
import { Game } from 'fraxel'

Game.blurred.on(() => {
  console.log('Game loop stopped ‚Äî tab lost focus')
  showPauseOverlay()
})
```

<Note>
The game loop stops on blur. When the browser regains focus, Fraxel calls
`Game.play()` automatically if the loop is not running. Set `pauseOnBlur` if you
also want `Game.isPaused()` to become `true` on blur.
</Note>

## Game.focused

An `Event` that fires whenever the browser window gains focus.

```ts
Game.focused: Event<[], 'focus'>
```

## TestOptions

Debug rendering options passed to `Game.setup` or `<GameRoot testOptions={...}>`.

```ts
interface TestOptions {
  showColliders:  boolean  // Draw collider shapes ‚Äî default false
  showRayCasts:   boolean  // Draw ray cast lines ‚Äî default false
  showClickables: boolean  // Draw clickable areas ‚Äî default false
}
```

All properties default to `false`. Enable them during development to visualise invisible game objects:

```ts
Game.setup({
  width: 320, height: 180, root,
  testOptions: {
    showColliders:  true,
    showRayCasts:   true,
    showClickables: true,
  },
})
```
