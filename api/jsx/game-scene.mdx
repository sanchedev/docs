`<Game>` and `<Scene>` are the two JSX components that define the structure of a Fraxel application. They are not rendered as DOM elements or canvas nodes — they are configuration declarations that `createGame` reads to wire up the engine, canvas, and scene manager.

---

## createGame

Bootstraps the entire Fraxel runtime from a `<Game>` JSX tree and returns `GameControls` for lifecycle management.

```ts
import { createGame } from 'fraxel/jsx'

createGame(jsx: Fraxel.Node, root: HTMLElement): GameControls
```

<ParamField path="jsx" type="Fraxel.Node" required>
  Must be a `<Game>` element containing one or more `<Scene>` children.
</ParamField>
<ParamField path="root" type="HTMLElement" required>
  The DOM element that will become the parent of the generated `<canvas>`. Passing `null` throws `MissingGameRootError`.
</ParamField>

**Returns** [`GameControls`](#gamecontrols).

**What `createGame` does:**
1. Reads `width`, `height`, and `testOptions` from `<Game>` props and calls `Game.setup()`.
2. Iterates `<Scene>` children and registers each with `Game.sceneManager`.
3. Calls `Game.sceneManager.setScene(defaultScene)` to load the first scene.
4. Returns a `GameControls` object.

```tsx
import { createGame, Game, Scene } from 'fraxel/jsx'

const game = createGame(
  <Game width={320} height={180} defaultScene="main">
    <Scene name="main"  component={() => import('./scenes/main.js')} />
    <Scene name="level2" component={() => import('./scenes/level2.js')} />
    <Scene name="menu"  component={MenuScene} />
  </Game>,
  document.querySelector<HTMLElement>('#root')!
)

game.play()
```

## \<Game\>

The root configuration component. Must be the top-level element passed to `createGame`. Does not produce a canvas node by itself — `createGame` reads its props and calls `Game.setup()`.

```ts
interface GameOptions {
  width:        number
  height:       number
  defaultScene: string
  testOptions?: Partial<TestOptions>
  theme?:       Theme
  children:     Fraxel.Node   // one or more <Scene> elements
}
```

<ParamField path="width" type="number" required>
  Logical canvas width in pixels. The canvas will be scaled by the browser's device-pixel ratio automatically.
</ParamField>
<ParamField path="height" type="number" required>
  Logical canvas height in pixels.
</ParamField>
<ParamField path="defaultScene" type="string" required>
  Name of the `<Scene>` to load first. Must match the `name` prop of one of the children.
</ParamField>
<ParamField path="pauseOnBlur" type="boolean">
  When `true`, the game pauses on browser blur (tab switch, minimize) and requires a manual `play()` call to resume. Defaults to `false`.
</ParamField>
<ParamField path="testOptions" type="Partial<TestOptions>">
  Enable debug overlays during development. See [`TestOptions`](#testoptions).
</ParamField>
<ParamField path="children" type="Fraxel.Node" required>
  One or more `<Scene>` elements defining all registered scenes.
</ParamField>

```tsx
<Game
  width={480}
  height={270}
  defaultScene="menu"
  testOptions={{ showColliders: true }}
>
  <Scene name="menu"   component={MenuScene} />
  <Scene name="level1" component={() => import('./level1.js')} />
</Game>
```

## \<Scene\>

Declares a named scene. Used exclusively as a direct child of `<Game>`.

```ts
interface SceneOptions {
  name:      string
  component: SceneComponent
}
```

<ParamField path="name" type="string" required>
  Unique identifier for the scene. Passed to `game.changeScene()`, `game.preloadScene()`, and `GameControls.changeScene()`.
</ParamField>
<ParamField path="component" type="SceneComponent" required>
  A function that returns the scene's root JSX node. Can be lazy (async import) or eagerly defined. See [`SceneComponent`](#scenecomponent).
</ParamField>

<CodeGroup>
```tsx Eager (no code splitting)
import { MainScene } from './scenes/main.js'

<Scene name="main" component={MainScene} />
```
```tsx Lazy (dynamic import)
<Scene name="main" component={() => import('./scenes/main.js')} />
```
```tsx Inline async
<Scene
  name="intro"
  component={async () => {
    const [BG] = await loadBatchAsset('texture', ['/assets/intro.png'])
    return () => <sprite textureId={BG} />
  }}
/>
```
</CodeGroup>

## SceneComponent

The type accepted by the `component` prop of `<Scene>`.

```ts
type SceneComponent =
  | (() => Fraxel.Node)                              // eager — returns JSX directly
  | (() => Promise<() => Fraxel.Node>)               // lazy — async, returns a factory
  | (() => Promise<{ default: () => Fraxel.Node }>)  // lazy — ES module default export
```

All three forms are handled by `createGame`. The most common patterns:

| Form | When to use |
|---|---|
| `() => JSX` | Small scene with no async asset loading needed. |
| `() => import('./scene.js')` | Code-split scene module with a default export that is the component. |
| `async () => { await load(); return () => JSX }` | Scene that loads assets before rendering. |

<Note>
The `SceneComponent` function must ultimately return (or resolve to a function that returns) **exactly one root node**. Multiple root nodes will throw `InvalidSceneComponentError`.
</Note>

## GameControls

The object returned by `createGame`.

```ts
interface GameControls {
  play():                     void
  pause():                    void
  destroy():                  void
  changeScene(name: string):  Promise<void>
  preloadScene(name: string): Promise<() => void>
  getSize():                  Vector2
  paused:                     SignalGetter<boolean>
}
```

| Property | Type | Description |
|---|---|---|
| `paused` | `SignalGetter<boolean>` | Reactive signal that is `true` when the game loop is paused. Read with `game.paused()`. |
| `play()` | `() => void` | Starts or resumes the game loop. Acquires a screen wake lock on supported platforms. |
| `pause()` | `() => void` | Pauses the game loop. The canvas freezes on the last rendered frame. |
| `destroy()` | `() => void` | Stops the loop, clears the active scene, destroys input listeners, and resets state. |
| `changeScene(name)` | `(name: string) => Promise<void>` | Destroys the active scene, loads the named scene, resolves when complete. |
| `preloadScene(name)` | `(name: string) => Promise<() => void>` | Loads a scene in the background. Resolves with a function; call it to switch to the scene instantly. |
| `getSize()` | `() => Vector2` | Returns a `Vector2` equal to `{ x: width, y: height }` as configured in `<Game>`. |

```ts
const game = createGame(<Game ...>, root)
game.play()

// Switch after a delay
setTimeout(async () => {
  await game.changeScene('level2')
}, 3000)

// Preload then switch on button press
const activate = await game.preloadScene('boss')
button.onClick = () => activate()
```

## TestOptions

Debug overlay configuration used to visualise invisible game objects during development.

```ts
interface TestOptions {
  showColliders:  boolean   // outline all collider shapes
  showRayCasts:   boolean   // draw ray cast lines and hit indicators
  showClickables: boolean   // outline all clickable areas
}
```

All default to `false`. Pass partial objects — only the keys you set will change.

```tsx
<Game
  width={320}
  height={180}
  defaultScene="main"
  testOptions={{
    showColliders: true,
    showClickables: true,
  }}
>
  ...
</Game>
```
