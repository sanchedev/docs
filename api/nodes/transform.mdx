Use `<transform>` to position child nodes in world space. It has no visual representation of its own — it exists purely as a coordinate-space container. Any node that needs a world position (a player, an enemy, a UI group) should live inside a `<transform>`. Because transforms nest, a child's position is always relative to its parent's `position`.

```tsx
import { useNode, useEvent } from 'fraxel/hooks'
import { PrimaryNode } from 'fraxel'

const PLAYER_TEX = await loadTexture('/assets/player.png')

function Player() {
  const body = useNode(PrimaryNode.Transform)

  // Move right at 80 px/s
  useEvent(body, 'updated', (delta) => {
    body.node.position.x += delta * 80
  })

  return (
    <transform ref={body} position={[100, 200]}>
      <sprite textureId={PLAYER_TEX} />
    </transform>
  )
}
```

## Props

`<transform>` inherits all props from the shared **Node2D** base and adds none of its own.

<ParamField path="position" type="VectorLike | SignalGetter<VectorLike>">
  World-space position of this node relative to its parent. Accepts an `[x, y]`
  tuple, an `{ x, y }` object, or a reactive `SignalGetter`. Defaults to
  `[0, 0]`.

  ```tsx
  <transform position={[100, 50]} />
  <transform position={() => [playerX(), playerY()]} />
  ```
</ParamField>

<ParamField path="id" type="string | symbol">
  Optional identifier for this node. Use with `useChild()` or `node.child()` to
  look up nested nodes by path. Must match `[a-zA-Z][a-zA-Z0-9-_]*` when a
  string.

  ```tsx
  <transform id="container">
    <transform id="inner" />
  </transform>
  ```
</ParamField>

<ParamField path="zIndex" type="number">
  Drawing order relative to siblings. Higher values render on top. Defaults to
  `0`.
</ParamField>

<ParamField path="deltaIncrease" type="number">
  Time-scale multiplier applied to this node and all its children. `0.5` = half
  speed, `2` = double speed. Defaults to `1`.

  ```tsx
  {/* Slow-motion subtree */}
  <transform deltaIncrease={0.25}>
    <sprite textureId={EFFECT_TEX} />
  </transform>
  ```
</ParamField>

<ParamField path="ref" type="NodeReference<PrimaryNode.Transform>">
  Attach a node reference created with `useNode(PrimaryNode.Transform)` to
  access the instance imperatively.
</ParamField>

## Computed properties

| Property | Type | Description |
|---|---|---|
| `globalPosition` | `Vector2` (read/write) | Absolute world-space position, accounting for all ancestor offsets. Setting it back-calculates the required local `position`. |
| `parent` | `Node \| undefined` | The parent node in the scene tree. |
| `children` | `Node[]` | All immediate child nodes. |

## Events

Listen to these with `useEvent(ref, eventName, callback)`.

| Event name | Callback signature | Description |
|---|---|---|
| `started` | `() => void` | Fires once when the node finishes its `start()` lifecycle. |
| `updated` | `(delta: number) => void` | Fires every frame during the update cycle. `delta` is elapsed seconds since the last frame. |
| `destroyed` | `() => void` | Fires when the node is removed from the scene tree. |

## Methods

| Method | Signature | Description |
|---|---|---|
| `addChild` | `(...nodes: Node[]) => void` | Dynamically attach child nodes at runtime. |
| `removeChild` | `(node: Node) => void` | Detach a child node without destroying it. |
| `child` | `({ path, type }) => Node` | Find a descendant by ID path and node type. Throws if not found. |
| `destroy` | `() => void` | Remove this node and all its children from the scene. |

```tsx
// Traverse the hierarchy at runtime
const inner = body.node.child({ path: ['container', 'inner'], type: PrimaryNode.Sprite })
```

<Tip>
  Nest transforms freely — a weapon node attached to a character transform
  automatically inherits the character's position. Use `globalPosition` when
  you need the absolute world coordinate regardless of depth.
</Tip>
