Audio is a critical part of game feel. A jump sounds heavier with a thud on landing. An enemy feels more threatening when it growls. Fraxel's audio system is built on the Web Audio API and gives you a simple, declarative interface: import a sound to get a symbol ID, attach an `<audio>` node wherever you need playback, and call `play()`.

## Importing sounds

In projects created with `create-fraxel`, import static sounds with `?sound`. The Vite plugin decodes each file before the module exports and gives you a `symbol` ID.

```ts src/assets.ts
export { default as SHOOT_SFX } from './assets/sounds/shoot.mp3?sound'
export { default as JUMP_SFX } from './assets/sounds/jump.mp3?sound'
export { default as BG_MUSIC } from './assets/sounds/music.mp3?sound'
export { default as HIT_SFX } from './assets/sounds/hit.mp3?sound'
```

Use `loadSound(url)` only when the sound URL is not known at build time. In `0.1.0-alpha.6d`, runtime `loadSound` calls do not deduplicate by URL: calling `loadSound` with the same URL twice returns two different symbols.

<Note>
  `0.1.0-alpha.7` is planned to deduplicate sounds by URL. Until then, load
  shared sounds once in an assets module and import the symbol where needed.
</Note>

## The `<audio>` node

Place an `<audio>` node anywhere in your scene tree to enable sound playback for that component. Pass it the imported sound symbol as the `soundId` prop.

```tsx src/components/gun.tsx
import { useAudio, useClickable, useTrigger } from 'fraxel'
import { SHOOT_SFX, GUN_TEX } from '../assets.js'

export function Gun() {
  const audio = useAudio()
  const btn = useClickable()

  useTrigger(btn.clicked, () => {
    audio.play()
  })

  return (
    <sprite textureId={GUN_TEX}>
      <audio ref={audio} soundId={SHOOT_SFX} volume={0.8} />
      <clickable ref={btn} size={[24, 12]} />
    </sprite>
  )
}
```

### Props

| Prop              | Type      | Default | Description                                             |
| ----------------- | --------- | ------- | ------------------------------------------------------- |
| `soundId`         | `symbol`  | -       | Sound ID from a `?sound` import or `loadSound`           |
| `loop`            | `boolean` | `false` | Loop the audio indefinitely                             |
| `volume`          | `number`  | `1`     | Playback volume from `0` to `1`                         |
| `playbackRate`    | `number`  | `1`     | Speed factor — `2` plays at double speed                |
| `persistUntilEnd` | `boolean` | `false` | Delay node destruction until the sound finishes playing |

### Methods

Call these on `audio.node` inside event handlers or `useMount`:

| Method          | Description                                                     |
| --------------- | --------------------------------------------------------------- |
| `play(offset?)` | Start or resume playback. Pass a start time in seconds to seek. |
| `pause()`       | Pause without resetting the playback position.                  |
| `stop()`        | Stop and reset position to the beginning.                       |

### Events

| Event   | Callback        | Description                                    |
| ------- | --------------- | ---------------------------------------------- |
| `ended` | `() => void`    | Fires when a non-looping sound reaches its end |
| `error` | `(err) => void` | Fires if playback fails                        |

## Background music

For looping background music that plays as soon as the scene starts, use `useMount` to trigger playback after the node has entered the scene.

```tsx src/components/background-music.tsx
import { useAudio, useMount } from 'fraxel'
import { BG_MUSIC } from '../assets.js'

export function BackgroundMusic() {
  const audio = useAudio()

  useMount(() => {
    audio.play()
  })

  return <audio ref={audio} soundId={BG_MUSIC} loop volume={0.35} />
}
```

## Browser autoplay policy

Browsers block audio from playing before the user has interacted with the page. Fraxel's `AudioContext` is created lazily — it exists as soon as `loadSound` is called, but it may start in a `suspended` state.

To handle this, resume the context inside a user gesture callback (a click, tap, or keypress):

```tsx src/components/start-screen.tsx
import { useClickable, useScene, useTrigger } from 'fraxel'
import { getAudioContext } from 'fraxel'

export function StartScreen() {
  const btn = useClickable()
  const scene = useScene()

  useTrigger(btn.clicked, async () => {
    const ctx = getAudioContext()
    if (ctx.state === 'suspended') {
      await ctx.resume()
    }
    // Now safe to start music
    scene.change('main')
  })

  return (
    <sprite textureId={START_SCREEN_TEX}>
      <clickable ref={btn} size={[192, 112]} />
    </sprite>
  )
}
```

<Note>
  If your game starts with a title screen or any other click-to-start flow, the
  user gesture to dismiss it also satisfies the autoplay policy. Call
  `ctx.resume()` inside that gesture handler and audio will be ready to play as
  soon as the player enters the game.
</Note>

## Reactive playback with `useAudio`

The `useAudio` derived hook gives you reactive `playing` state and control methods without manual wiring.

```tsx
import { useAudio, useClickable, useTrigger } from 'fraxel'

function ToggleMusic() {
  const audio = useAudio()
  const btn = useClickable()

  useTrigger(btn.clicked, () => {
    audio.playing() ? audio.pause() : audio.play()
  })

  return (
    <sprite textureId={BTN_TEX}>
      <clickable ref={btn} size={[32, 32]} />
      <audio ref={audio} soundId={BG_MUSIC} loop volume={0.4} />
    </sprite>
  )
}
```

## Preloading with `loadBatch`

For sounds whose URLs are only known at runtime, use `loadBatchAsset` to load a group of assets in parallel with an optional progress callback.

```ts src/assets.ts
import { loadBatchAsset } from 'fraxel'

export const [SHOOT_SFX, JUMP_SFX, HIT_SFX, BG_MUSIC] = await loadBatchAsset(
  'sound',
  [
    '/assets/sounds/shoot.mp3',
    '/assets/sounds/jump.mp3',
    '/assets/sounds/hit.mp3',
    '/assets/sounds/music.mp3',
  ],
)
```

Batch loading is especially useful on the loading screen - you can display a progress bar while assets decode before calling `game.play()`.

## Complete audio setup example

```tsx src/main.tsx
import { createGame, GameRoot, SceneRoot } from 'fraxel'
import { useAudio, useMount } from 'fraxel'
import { getAudioContext } from 'fraxel'
import BG_MUSIC from './assets/sounds/music.mp3?sound'

function MainScene() {
  const music = useAudio()

  useMount(() => {
    const ctx = getAudioContext()
    if (ctx.state === 'suspended') ctx.resume()
    music.play()
  })

  return (
    <transform>
      <audio ref={music} soundId={BG_MUSIC} loop volume={0.3} />
    </transform>
  )
}

const game = createGame(
  <GameRoot width={320} height={180} defaultScene="main">
    <SceneRoot name="main" component={MainScene} />
  </GameRoot>,
  document.querySelector('#app')!,
)

game.play()
```
