For new games, use [Quickstart](/getting-started/quickstart) and create a project with `create-fraxel`. That is the recommended path and the context assumed by the guides.

Use this page when you already have a Vite app or when you intentionally want to embed a small interactive Fraxel scene inside another page.

## Existing Vite project

Install the runtime and the Vite plugin:

<CodeGroup>

```bash npm
npm install fraxel
npm install --save-dev @fraxel/vite-plugin
```

```bash pnpm
pnpm add fraxel
pnpm add -D @fraxel/vite-plugin
```

```bash yarn
yarn add fraxel
yarn add --dev @fraxel/vite-plugin
```

</CodeGroup>

Configure Vite:

```js vite.config.js
import { defineConfig } from 'vite'
import { fraxel } from '@fraxel/vite-plugin'

export default defineConfig({
  plugins: [fraxel()],
})
```

Configure TypeScript:

```json tsconfig.json
{
  "compilerOptions": {
    "target": "ES2022",
    "module": "ESNext",
    "moduleResolution": "Bundler",
    "strict": true,
    "jsx": "react-jsx",
    "jsxImportSource": "fraxel",
    "types": ["@fraxel/vite-plugin/client"]
  }
}
```

The plugin handles Fraxel's JSX runtime and lets you import static assets as Fraxel symbols:

```tsx
import PLAYER from './assets/player.png?texture'
import JUMP from './assets/jump.ogg?sound'
```

Plain imports without `?texture` or `?sound` use Vite's normal URL-string behavior.

## HTML root

Fraxel creates a `<canvas>` inside the element you pass to `createGame()`.

```html index.html
<div id="app"></div>
<script type="module" src="/src/main.tsx"></script>
```

```tsx src/main.tsx
import { createGame, GameRoot, SceneRoot } from 'fraxel'
import './style.css'

const game = createGame(
  <GameRoot width={480} height={360} defaultScene="main">
    <SceneRoot name="main" component={() => import('./scenes/main')} />
  </GameRoot>,
  document.querySelector('#app')!,
)

game.play()
```

## Embedded or small no-plugin setup

<Note>
  Using `fraxel` without `@fraxel/vite-plugin` is intended for small embedded
  games or custom integrations inside another page. It is not the recommended
  setup for full projects.
</Note>

If you skip the plugin, you must configure JSX yourself and load assets with runtime URLs:

```json tsconfig.json
{
  "compilerOptions": {
    "jsx": "react-jsx",
    "jsxImportSource": "fraxel"
  }
}
```

```ts
import { loadTexture } from 'fraxel'

const PLAYER = await loadTexture('/assets/player.png')
```

This works well for small interactive embeds where you do not need the full project workflow that `create-fraxel` and the Vite plugin provide.
