This page covers manual installation without the CLI. If you prefer to set up everything yourself — or need to add Fraxel to an existing project — follow the steps below.

## Install the package

<CodeGroup>

```bash npm
npm install fraxel
```

```bash yarn
yarn add fraxel
```

```bash pnpm
pnpm add fraxel
```

</CodeGroup>

Fraxel has no runtime dependencies. Everything — the renderer, reactivity system, physics, audio, and input — ships in the single `fraxel` package.

## Configure TypeScript

Add the following to your `tsconfig.json`. The critical options are `jsx: "react-jsx"` and `jsxImportSource: "fraxel"`, which tell TypeScript to compile JSX using Fraxel's runtime instead of React's.

```json tsconfig.json
{
  "compilerOptions": {
    "target": "ES2022",
    "module": "ESNext",
    "moduleResolution": "Bundler",
    "strict": true,
    "verbatimModuleSyntax": true,
    "jsx": "react-jsx",
    "jsxImportSource": "fraxel"
  }
}
```

<Note>
  `verbatimModuleSyntax` enforces that type-only imports use the `import type`
  syntax. You will need to write `import type {NodeReference} from 'fraxel'`
  instead of `import {NodeReference} from 'fraxel'` for types. This keeps the
  emitted JavaScript clean and avoids accidental runtime imports of type-only
  symbols.
</Note>

## Create your HTML entry point

Fraxel renders into a `<canvas>` element that it creates and mounts inside a container element in your HTML. Add a `div` with an `id` of `root` (or any ID you prefer) and a `<script>` tag pointing at your compiled entry file.

```html index.html
<!doctype html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>My Fraxel Game</title>
    <style>
      body {
        margin: 0;
        background: #000;
        display: flex;
        align-items: center;
        justify-content: center;
        height: 100vh;
      }
      canvas {
        image-rendering: pixelated;
      }
    </style>
  </head>
  <body>
    <div id="root"></div>
    <script type="module" src="/src/main.tsx"></script>
  </body>
</html>
```

<Tip>
  Setting `image-rendering: pixelated` on the canvas prevents the browser from
  blurring low-resolution pixel-art games when the canvas is scaled up via CSS.
</Tip>

## Import paths

Everything is exported from a single entry point — `'fraxel'`:

```tsx main.tsx
import { createGame, GameRoot, SceneRoot } from 'fraxel'
import { useSignal, useSprite, useTrigger } from 'fraxel'
import { loadTexture, shapes } from 'fraxel'
```

## Verify your setup

Create a minimal `src/main.tsx` and run your bundler. If TypeScript compiles without errors and you see a black canvas in the browser, your setup is correct.

```tsx src/main.tsx
import { createGame, GameRoot, SceneRoot } from 'fraxel'

const scene = () => <transform />

const game = createGame(
  <GameRoot width={320} height={180} defaultScene='main'>
    <SceneRoot name='main' component={scene} />
  </GameRoot>,
  document.querySelector('#root')!,
)

game.play()
```

<Note>
  Fraxel uses `top-level await` for asset loading in most projects. Make sure your bundler targets ES2022 or later and your `<script>` tag uses `type="module"`.
</Note>
