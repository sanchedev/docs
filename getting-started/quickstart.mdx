The recommended way to start any new Fraxel project is the `create-fraxel` CLI. It scaffolds a Vite project with TypeScript, `@fraxel/vite-plugin`, the Fraxel JSX runtime, and a starter scene ready to edit.

The rest of Getting Started and Guides assume this setup.

<Note>
  The `create-fraxel` CLI requires Node.js `20.0.0` or newer. The `fraxel`
  runtime package itself supports Node.js `18` or newer for package tooling.
</Note>

## Scaffold a project

<CodeGroup>

```bash npm
npx create-fraxel my-game --template empty
```

```bash pnpm
pnpm create fraxel my-game --template empty
```

```bash yarn
yarn create fraxel my-game --template empty
```

</CodeGroup>

The CLI will guide you through:

- **Choose a template** — `Empty`, `Platformer`, or `Top-down`
- **Install dependencies?** — optionally run `npm install` automatically

```
┌  🎮 create-fraxel
│
◇  Choose a template
│  Empty
│
◇  Install dependencies?
│  No
│
◇  Creating project...
│
└  ✨ Project ready!

cd my-game
npm install
npm run dev
```

The CLI creates a `my-game/` directory with the following structure:

```
my-game/
├── src/
│   ├── main.tsx
│   ├── scenes/
│   │   └── main.tsx
│   └── style.css
├── index.html
├── package.json
├── tsconfig.json
└── vite.config.js
```

## What the template includes

**Vite + Fraxel plugin** — `vite.config.js` comes pre-configured with `@fraxel/vite-plugin`. You do not need to set `esbuild.jsxImportSource` or any JSX options manually — the plugin handles it.

```js vite.config.js
import { defineConfig } from 'vite'
import { fraxel } from '@fraxel/vite-plugin'

export default defineConfig({
  plugins: [fraxel()],
})
```

**TypeScript configured** — `tsconfig.json` includes the Fraxel JSX settings and client types for asset imports:

```json tsconfig.json
{
  "compilerOptions": {
    "target": "ES2022",
    "module": "ESNext",
    "moduleResolution": "Bundler",
    "strict": true,
    "jsx": "react-jsx",
    "jsxImportSource": "fraxel",
    "types": ["@fraxel/vite-plugin/client"]
  }
}
```

**Asset imports** — with the plugin, add `?texture` or `?sound` to import a Fraxel texture or sound symbol at build time:

```tsx src/scenes/main.tsx
import PLAYER from '../assets/sprites/player.png?texture'

export function MainScene() {
  return (
    <transform>
      <sprite textureId={PLAYER} />
    </transform>
  )
}
```

The query param selects `loadTexture` or `loadSound`. The imported value is a `symbol` that you pass directly to node props.

**Supported asset formats:**

| Type     | Extensions                                                                                |
| -------- | ----------------------------------------------------------------------------------------- |
| Textures | `.png`, `.jpg`, `.jpeg`, `.gif`, `.svg`, `.webp`, `.avif`, `.bmp`, `.ico`, `.cur`, `.jxl` |
| Sounds   | `.ogg`, `.mp3`, `.wav`, `.flac`, `.aac`, `.opus`, `.m4a`                                  |

## Run the dev server

<CodeGroup>

```bash npm
cd my-game && npm run dev
```

```bash pnpm
cd my-game && pnpm dev
```

```bash yarn
cd my-game && yarn dev
```

</CodeGroup>

Vite starts a dev server with standard development reload behavior.

## Loading runtime assets from `public/`

The template also supports loading assets from `public/` with URL strings. Use this for dynamically constructed paths or assets loaded at runtime:

```ts
import { loadTexture } from 'fraxel'

const texture = await loadTexture('/assets/sprites/player.png')
```

Both patterns work in the same project. In guides, static assets use `?texture` and `?sound`; runtime URLs use `loadTexture` and `loadSound`.
