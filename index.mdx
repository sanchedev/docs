## What makes Fraxel different

Fraxel is not a wrapper around an existing engine or a React renderer for canvas. JSX elements like `<sprite>`, `<collider>`, and `<body>` compile directly into engine node instances. The result is a declarative, component-driven authoring experience that runs at native canvas speed.

## Get up and running

<Steps>
  <Step title="Install the package">
    Add Fraxel to your project with your preferred package manager.

    ```bash
    npm install fraxel
    ```

  </Step>
  <Step title="Configure the JSX runtime">
    Tell TypeScript to use Fraxel's JSX transform in `tsconfig.json`.

    ```json tsconfig.json
    {
      "compilerOptions": {
        "jsx": "react-jsx",
        "jsxImportSource": "fraxel"
      }
    }
    ```

  </Step>
  <Step title="Write your first scene">
    Create a `main.tsx` file, import a texture, define a scene, and start the game loop.

    ```tsx main.tsx
    import { createGame, GameRoot, SceneRoot } from 'fraxel'
    import PLAYER from './assets/player.png?texture'

    const scene = () => <sprite textureId={PLAYER} position={[80, 50]} />

    const game = createGame(
      <GameRoot width={320} height={180} defaultScene="main">
        <SceneRoot name="main" component={scene} />
      </GameRoot>,
      document.querySelector('#app')!,
    )

    game.play()
    ```

  </Step>
  <Step title="Add reactivity">
    Use `useSignal` to create reactive state. Pass a signal getter directly as a JSX prop — Fraxel tracks the dependency automatically.

    ```tsx
    import { useSignal } from 'fraxel'

    function Player() {
      const [health, setHealth] = useSignal(100)

      return (
        <sprite
          textureId={PLAYER}
          grayscale={() => (health() <= 0 ? 1 : 0)}
          brightness={() => 0.5 + health() / 200}
        />
      )
    }
    ```

  </Step>
</Steps>
